隱私權政策

最後更新日期：2026 年 1 月 6 日

「更好的健保雲端 2.0」（以下簡稱「本擴充功能」）尊重並保護使用者的隱私權。本政策說明我們如何處理您的資料。

1. 資料收集與儲存

1.1 本地儲存的資料

本擴充功能使用 Chrome Storage API 在您的瀏覽器本地儲存以下資料：

  • 使用者偏好設定（顯示選項、追蹤天數等）
  • AI 服務的 API Key（若您選擇使用 GAI 功能）
  • GAI 對話歷史（僅存於本地，不同步至雲端）
重要說明：您的 API Key 僅儲存於本地瀏覽器中，不會傳輸至任何第三方伺服器（除了您選擇的 AI 服務提供者）。

1.2 不收集的資料

本擴充功能不會：

  • 收集或傳送任何資料至本擴充功能的開發者
  • 追蹤您的瀏覽行為
  • 儲存病患資料於任何外部伺服器
  • 分享您的資料給廣告商或資料仲介

2. GAI 智慧分析功能

注意：GAI 功能為選用功能，預設為關閉。啟用前請詳閱以下說明。

2.1 資料傳輸

當您啟用並使用 GAI 智慧分析功能時，系統會將您選擇的醫療資料類型傳送至您所選擇的 AI 服務提供者進行分析：

AI 服務提供者 API 端點 隱私權政策
Cerebras (推薦) api.cerebras.ai 連結
Groq api.groq.com 連結
Google Gemini generativelanguage.googleapis.com 連結
OpenAI api.openai.com 連結

2.2 自動去個資化保護

隱私保護機制：本擴充功能內建自動去個資化（De-identification）系統，在傳送影像報告等資料至 AI 服務前，會自動遮罩以下敏感資訊：
  • 病患資訊：姓名、病歷號碼、病床號
  • 醫療人員資訊：醫師姓名、放射師姓名、護理師姓名、各類醫療人員證照號碼
  • 系統編號：醫囑單號、影像號、內視鏡編號

支援多種醫院報告格式的深度優化處理。

2.3 選擇性資料傳輸

您可以自由選擇要傳送哪些類型的醫療資料進行分析（共 10 種類型），未選擇的資料類型不會被傳送。

2.4 免費方案警告

重要警告：部分 AI 服務的免費方案可能會使用您提交的內容來改進其 AI 模型。若您處理高度敏感的醫療資料，建議：
  • 使用付費方案（通常不會用於模型訓練）
  • 避免傳送可直接識別個人身份的資訊
  • 詳閱各 AI 服務提供者的資料使用政策

3. 權限說明

本擴充功能請求以下權限：

權限 用途
webRequest 攔截健保雲端系統的 API 回應以擷取醫療資料
storage 儲存使用者設定和 API Key 於本地
scripting 在健保雲端網頁注入內容腳本
clipboardWrite 支援複製功能
host_permissions 存取健保雲端網站及 AI API 端點

4. 資料控制權

您隨時可以：

  • 關閉 GAI 功能：在設定中停用「啟用 GAI 智慧分析側邊欄」
  • 刪除 API Key：在設定中清除已儲存的 API Key
  • 清除對話歷史：使用 Chat 分頁的「清除歷史」按鈕
  • 重設所有設定：使用「重設為預設值」功能或移除擴充功能

5. 資料安全

  • 本擴充功能僅在 medcloud2.nhi.gov.tw 網域運作
  • 所有本地資料使用 Chrome 的安全儲存機制
  • API 通訊皆使用 HTTPS 加密傳輸
  • 本擴充功能為開源專案，程式碼可供檢視

6. 第三方服務

本擴充功能可能連線至以下第三方服務：

  • drugtw.com：查詢藥品圖片（不傳送病患資料）
  • AI 服務提供者：僅在您啟用 GAI 功能並主動觸發分析時

7. 政策更新

本隱私權政策可能會不定期更新。重大變更將透過擴充功能更新說明或 GitHub 專案頁面公告。

8. 聯絡方式

如有任何隱私相關問題，請透過以下方式聯繫：
